# 项目总览

## 这个项目本质是什么

GitHub AI Project Analyzer 本质上是一个 AI 驱动的项目筛选与学习推荐工具。

它不是简单地把 GitHub 搜索结果打印出来，而是把搜索结果交给 AI 分析，再结合评分规则生成推荐排名。

可以从三个角度理解它：

## 1. 一个 AI 项目筛选器

用户输入关键词，例如：

```text
ai agent
rag
llm app
```

程序会通过 GitHub API 搜索相关项目，并提取项目名称、简介、星标数、issue 数、语言、更新时间、许可证等信息。

它解决的是：

> GitHub 项目太多，初学者不知道先看哪个。

## 2. 一个学习推荐系统

程序不会只看星标数。

它会结合：

- 项目星标数；
- issue 数量；
- 项目简介是否完整；
- DeepSeek 对学习难度、学习价值、第一步建议、风险提醒的分析；
- 内置评分规则。

最后生成：

- `metadata_score`
- `ai_learning_score`
- `final_score`
- `final_recommendation`

这些结果会进入 Markdown 推荐报告，方便用户阅读和比较。

## 3. 一个 LLM 驱动的决策工具

这个项目的关键点不是“请求了一个接口”，而是把 LLM 放在一个决策流程中：

```text
搜索数据 → AI 理解 → JSON 结构化 → 字段校验 → 评分 → 排名 → 推荐
```

也就是说，DeepSeek 不是随便聊天，而是在帮助程序做结构化判断。

## 不是爬虫，不是脚本，是 AI 决策工具

这个项目不是爬虫。

原因：

- 它没有批量抓取网页 HTML；
- 它使用的是 GitHub 官方 Search API；
- 它不会绕过网站规则抓取数据。

这个项目也不只是普通脚本。

普通脚本通常是：

```text
输入 → 固定规则处理 → 输出
```

而这个项目是：

```text
输入搜索条件
    ↓
GitHub API 获取真实项目
    ↓
DeepSeek 理解项目学习价值
    ↓
Python 校验和评分
    ↓
输出推荐排名
```

它更接近一个小型 AI 决策工具：把外部数据、AI 判断、结构化输出和排序规则组合在一起，帮助用户做选择。

## 当前真实能力

当前已经实现：

- 命令行参数输入；
- 交互式输入；
- GitHub API 搜索；
- DeepSeek API 分析；
- JSON 解析；
- 字段完整性校验；
- 成功和失败结果分离；
- 学习适合度评分；
- 最终排名；
- Markdown 报告生成。

当前未实现：

- 自动读取 GitHub README 全文；
- GitHub Token 配置；
- Web 页面；
- 数据库；
- 后台定时任务；
- 自动发送通知。

## 适合谁使用

这个项目适合：

- Python 初学者；
- AI Agent 学习者；
- 想从 GitHub 项目中找练习对象的人；
- 想学习“API + LLM + JSON + CLI + 报告生成”完整流程的人。

## 适合作为 GitHub 项目的原因

它有清晰的输入、处理流程和输出：

```text
输入：关键词、语言、最低星标、返回数量
处理：GitHub 搜索 + DeepSeek 分析 + Python 评分
输出：JSON 结果 + Markdown 推荐报告
```

这让它不只是学习笔记，而是一个可以运行、可以解释、可以继续升级的小工具。
